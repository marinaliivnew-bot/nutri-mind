# План доработок NutriMind / Nutri Intelligence Agent

Дата: 26 июня 2026

**Статус документа:** завершен 26 июня 2026. Этот файл остается архивом продуктово-визуальной переработки NutriMind / Nutri Intelligence Agent: позиционирование, приватность, кабинеты, диалог, связность демо, безопасные формулировки и портфельный стиль доведены до состояния портфолио-прототипа.

**Дальнейшая разработка:** рабочая логика прототипа, нормализация локальной модели данных, связность кейсов, импорт/экспорт демо, MVP-архитектура и реальный AI-черновик ведутся в [`NUTRI_AGENT_WORKING_PROTOTYPE_PLAN.md`](NUTRI_AGENT_WORKING_PROTOTYPE_PLAN.md).

## 1. Цель доработки

Сделать из текущего Mimo-прототипа цельный портфолио-кейс и понятный продуктовый черновик SaaS-платформы для нутрициологов и специалистов функциональной медицины: кабинет, где нутрициолог ведет диагностику, клиент заполняет материалы, а ИИ помогает структурировать данные, гипотезы и черновики рекомендаций.

Ключевая позиция продукта:

> Нутрициолог принимает решения. ИИ помогает быстрее собрать картину, увидеть связи, подготовить черновик и объяснить рекомендации клиенту понятным языком.

## 1.1. Исходная продуктовая рамка из документов

После добавления исходников в проект нужно опираться не только на Mimo-прототип, но и на документы:

- `Nutri Agent исходники/Продвижение/Saas/описание продукта и лендинга.docx`
- `Nutri Agent исходники/Продвижение/Nutri Intelligence Agent кейс.docx`

Из них следует, что исходная идея шире, чем один GPT-агент:

- `NutriMind` — публичное название SaaS-платформы, "умного офиса" для нутрициолога.
- `Nutri Intelligence Agent` — название интеллектуального модуля/ассистента внутри платформы и хорошее название для технического кейса.
- Главная боль: нутрициолог тратит часы на ручную сверку анализов, дневников, жалоб, УЗИ, генетики и разных лабораторных форматов.
- Главная ценность: единое рабочее пространство, карта здоровья, системные связи, черновик клиентского отчета и протокола.
- Главный принцип доверия: продукт создан нутрициологом для нутрициологов и не заменяет специалиста.

Рекомендуемая архитектура названия:

- **Продукт / SaaS:** `NutriMind`
- **ИИ-модуль внутри продукта:** `Nutri Intelligence Agent`
- **Портфолио-кейс:** `NutriMind: SaaS-платформа с Nutri Intelligence Agent`
- **Короткий слоган:** "Ваш умный офис для нутрициологии"

Почему так:

- `NutriMind` короче, легче запоминается и лучше работает как бренд SaaS-продукта.
- `Nutri Intelligence Agent` звучит точнее для внутреннего движка, GPT-агента, демонстрации логики и технической части.
- Разделение названий помогает не смешивать витрину продукта, рабочий кабинет и сам ИИ-ассистент.

Перед публичным запуском стоит отдельно проверить название `NutriMind` на занятость доменов, соцсетей и товарных знаков. Для портфолио-прототипа его можно использовать как рабочее название.

## 3. Главные проблемы текущего черновика

### 3.1. Главная страница смешивает аудитории

Сейчас главный экран говорит в основном с нутрициологом: "ИИ-офис для нутрициолога", "за тебя", "экономит часы", "видит связи". Если клиент попадет на эту страницу, может возникнуть ощущение, что ИИ заменяет специалиста.

Нужно:

- Сделать общий нейтральный вход для двух ролей.
- Показать продукт как пространство совместной работы.
- Профессиональные УТП вынести в карточку/раздел для нутрициолога.
- Клиенту показывать язык заботы, удобства и прозрачности.

### 3.2. Роль нутрициолога должна быть сильнее

В текущем тексте местами звучит так, будто агент "делает за специалиста". Для доверия и юридической безопасности лучше закрепить:

- ИИ формирует черновик.
- Нутрициолог проверяет, корректирует, принимает или отклоняет гипотезы.
- Клиент получает рекомендации от нутрициолога, а не от ИИ напрямую.

### 3.3. Приватность не доведена до конца

Демо-клиенты обезличены, но форма "Добавить клиента" снова собирает имя и фамилию и добавляет реальное имя в список.

Нужно:

- В портфолио-версии использовать "Кейс #1001", "Кейс #1002" или псевдонимы.
- Убрать обязательные ФИО из формы добавления клиента.
- Добавить статус приватности: "демо-данные", "локально", "без реальных персональных данных".
- Реальное хранение персональных данных оставить как будущий production-блок.

### 3.4. Диалог пока не стал главным опытом

Диалог есть, но выглядит как один из табов. При этом именно диалоговый режим - главная фича продукта.

Нужно:

- Сделать диалог стартовым рабочим экраном нутрициолога.
- Встроить в диалог этапы: сбор данных, таблица параметров, отклонения, гипотезы, рекомендации, клиентская версия отчета.
- Добавить возможность комментировать гипотезы: принять, отклонить, уточнить, запросить источник.
- Показывать ссылки на источники/страницы как часть ответа агента.

### 3.5. Функции пока имитируются

Сейчас часть функций только показывает уведомления:

- "Сохранить" не сохраняет данные.
- "Экспорт" не создает файл.
- Загрузка файлов не влияет на анализ.
- Диалог не связан с выбранным клиентом, опросником, дневником и файлами.

Для портфолио это допустимо, но нужно честно оформить как прототип. Для следующей версии лучше реализовать хотя бы локальную связность.

- Проверить мобильную версию: главная, кабинет клиента, кабинет нутрициолога.

Результат: проект выглядит как зрелый SaaS-прототип и часть единого портфолио.

## 5. Рекомендуемый порядок работ

1. Зафиксировать нейминг: `NutriMind` как продукт, `Nutri Intelligence Agent` как ИИ-модуль.
2. Копирайт главной и разделение аудиторий.
3. Приватность: "клиенты" -> "кейсы", убрать ФИО из демо.
4. Клиентский кабинет: прогресс, нормальный дневник, статусы.
5. Диалог нутрициолога: поле ввода, этапы, действия по гипотезам.
6. Локальное сохранение и связность демо.
7. Безопасные медицинские формулировки.
8. Визуальный стиль под портфолио.

## 6. Техническое задание для следующей итерации

```
Доработать текущий прототип как SaaS-продукт NutriMind. NutriMind — это публичное название платформы, "умного офиса" для нутрициолога. Nutri Intelligence Agent — это ИИ-ассистент внутри кабинета нутрициолога.

1. Главная страница должна быть нейтральной для двух аудиторий: нутрициолога и клиента. Не создавай ощущения, что ИИ заменяет специалиста. Главная мысль: NutriMind собирает анализы, дневники, опросники и рекомендации в одном кабинете; нутрициолог ведет процесс, а Nutri Intelligence Agent помогает структурировать данные, находить связи и готовить черновики.

2. Разведи карточки входа:
   - Нутрициолог: единое рабочее пространство, база знаний, диалог с Nutri Intelligence Agent, карта здоровья, черновики отчетов и протоколов.
   - Клиент: опросник, дневник питания, анализы, статус материалов, рекомендации от нутрициолога понятным языком.

3. В демо-версии не используй реальные ФИО. Замени "Клиенты" на "Кейсы" или "Клиентские кейсы". Форма добавления должна создавать обезличенный кейс, а не человека с именем и фамилией.

4. Сделай диалог главным рабочим экраном нутрициолога. Добавь поле ввода, кнопки "Принять", "Уточнить", "Отклонить", "Показать источник". Гипотезы должны иметь статусы: подтверждено, предположение, недостаточно данных.

5. Клиентский кабинет должен быть спокойным и понятным: прогресс заполнения, опросник, дневник питания с формой внутри интерфейса, загрузка анализов, статус "отправлено нутрициологу", рекомендации "от вашего нутрициолога".

6. Замени медицински рискованные формулировки: не диагнозы, не лечение, не назначения. Используй "черновик интерпретации", "черновик рекомендаций", "для проверки специалистом".

7. В профессиональных УТП используй исходную идею из документов: экономия времени на ручной сверке, системные связи, разные форматы анализов и дневников, готовый клиентский отчет, черновик протокола, фокус специалиста на стратегии и клиенте.

8. Сохрани прототип как single-page HTML/CSS/JS без бэкенда, но добавь localStorage для состояния демо.
```

## 7. Критерии готовности следующей версии

- Клиент не видит профессиональные УТП, которые обесценивают роль нутрициолога.
- Нутрициолог видит сильный профессиональный инструмент.
- На публичных экранах продукт называется `NutriMind`; `Nutri Intelligence Agent` используется как название ассистента внутри кабинета.
- Демо не просит и не показывает реальные ФИО.
- Диалог выглядит как главный сценарий, а не декоративная вкладка.
- Клиентский кабинет понятен без инструкций.
- Есть честная маркировка демо-режима.
- Медицинские формулировки безопасны.
- После перезагрузки страницы базовое состояние сохраняется.
- Визуальный стиль ближе к общему портфолио.
